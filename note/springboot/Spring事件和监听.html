# Spring 事件和监听

Spring 中的事件机制基于观察者模式实现，事件发出时会通知该事件所有的监听者。

# 实现简单的事件和监听

首先我们来看看如何实现一个简单的事件和监听。下面的代码实现简单的演示了如何通过 jdk 相关的类和接口实现简单的事件和监听。

DemoEvent：事件类中主要继承 EventObject 类，可以自定义相关的属性。

public class DemoEvent extends EventObject {

    private String msg;
    
    public DemoEvent(Object source, String msg) {
        super(source);
        this.msg = msg;
    }

    public String getMsg() {
        return msg;
    }

    public void setMsg(String msg) {
        this.msg = msg;
    }

    @Override
    public String toString() {
        return "DemoEvent{" +
                "msg='" + msg + '\'' +
                ", source=" + source +
                '}';
    }
}

public class DemoListener implements EventListener {

    public void listener(DemoEvent demoEvent) {
        System.out.println("demoEvent = " + demoEvent);
    }

DemoEventPublisher：事件发布者类，持有所有监听事件的对象，在发布事件时通知到每一个监听该事件的监听者。

public class DemoEventPublisher {

    private List<DemoListener> listenerList = new ArrayList<>();

    public void monitor() {
        DemoEvent demoEvent = new DemoEvent(this, UUID.randomUUID().toString());
        publishEvent(demoEvent);
    }


    protected void publishEvent(DemoEvent event) {
        List<DemoListener> copyListenerList = new ArrayList<>(listenerList);
        for (DemoListener listener : copyListenerList) {
            listener.listener(event);
        }
    }

    public void addListener(DemoListener listener) {
        listenerList.add(listener);
    }

    public void removeListener(DemoListener listener) {
        listenerList.remove(listener);
    }

    public void removeAll() {
        listenerList.clear();
    }

    public static void main(String[] args) {
        DemoEventPublisher publisher = new DemoEventPublisher();
        publisher.addListener(new DemoListener());
        publisher.monitor();
    }
}

# 自定义事件

自定义事件需要继承继承 Spring 的 ApplicationEvent 类

@Slf4j
public class DemoEvent extends ApplicationEvent {
    public DemoEvent(String msg) {
        super(msg);
        log.info("event msg - {} - time - {}", msg, new Date());
    }
}

# 监听自定义事件

通过在方法上添加 @EventListener 注解实现，该注解以需要监听的事件作为参数，可以监听多个事件。@Async 表示异步执行，如果不加该注解是同步执行。

@Slf4j
@Component
public class DemoListener {

    @Async
    @EventListener(DemoEvent.class)
    public void listener(DemoEvent demoEvent) {
        String msg = (String) demoEvent.getSource();
        log.info("listener msg - {} - time - {}", msg, new Date());
    }
}

# Spring 中事件机制的使用

Spring 在所有的 Bean 初始化完成之后会发布一个 ContextRefreshedEvent 事件，这样就可以在项目启动的时候做一些初始或的工作。

@Slf4j
@Component
public class StartListener1 {

    @EventListener(ContextRefreshedEvent.class)
    public void listenerEvent(ContextRefreshedEvent contextRefreshedEvent) {
        if (Objects.isNull(contextRefreshedEvent.getApplicationContext().getParent())) {
            log.info("StartListener1 - {}", new Date());
        }
    }
}

@Slf4j
@Component
public class StartListener implements ApplicationListener<ContextRefreshedEvent> {
    @Override
    public void onApplicationEvent(ContextRefreshedEvent contextRefreshedEvent) {
        if (Objects.isNull(contextRefreshedEvent.getApplicationContext().getParent())) {
            log.info("StartListener - {}", new Date());
        }
    }
}

# 其他事件

事件 作用
ContextRefreshedEvent 当spring容器context刷新时触发
ContextStartedEvent 当spring容器context启动后触发
ContextStoppedEvent 当spring容器context停止时触发
ContextClosedEvent 当spring容器context关闭时触发，容器被关闭时，其管理的所有单例Bean都被销毁。
