# 2020年总结

​ 7月分刚毕业工作到现在已经有半年了，之前因为疫情原因没来实习，所以前半年都是在玩。当时一直想做毕设一直没能如愿，每天都是起床就玩，晚上伴着愧疚入睡，想着明天一定起来做毕设，到最后还是拖到了最后一个月才做。

# 工作

​ 先说说从7月低入职一直到现在的工作吧。入职前原本以为年末会有挺多事情要做的，入职后才发现做的事情基本都是对项目的修修补补（协助排查问题解决问题之类的），接手的项目上的需求基本很少，所以这半年来基本没怎么在工作上写过代码，唯一一次写过比较多的代码还是年末需要重构一个模块，不过也是简单的 CRUD 就可以解决，对自己的提升意义不大。

# 学习

​ 工作之外，这半年来也算是学习了一些东西。原本自己是没怎么有主动学习的想法的，但是入职之后真的算是没什么事做，可能是闲得慌，觉得自己再不找点事情做真的要顶不住了。

  • 首先是 Spring Cloud Alibaba，因为这个公司后续很多项目都会用到，所以先学习了这个，当时学这个之前我还特意先学了 Spring Cloud，到后面学完才发现这两个基本都没什么关系，虽然做的事情是一样的。
  • 之后就是 Redis 了，当时我是在学习消息队列的时候看到了一篇 Redis 面试文章写的很好之后才去学习的，当时对于 Redis 也就停留在了解阶段，看了这篇文章之后我就突然对 Redis 很感兴趣了，这片文章也算是我从不愿意主动学习转变成主动去学习的一个引子吧。学习 Redis 过程就不说了，断断续续学了好久，从自己网上找资料，到看别人准备好的课程，再到看书，这其中自己经历了很多，有好几次想放弃的想法（自己学一个东西学了很久之后突然发现只是学了冰山一角之后），不过幸好都坚持下来了。
  • Netty。学习 Netty 是因为当时学习到了 Redis 的 I/O 多路复用，当时一直不是很理解这个知识所以搜了很多文章看，在看 Reactor 模型的时候就看到了 Netty。这个是学 Netty 的原因之一，还有一个原因是发现自己平时写代码太依赖 Spring Boot 了，有一种想摆脱这种依赖的想法。从 nio，到一个简单的客户端服务端通信，再到看别人的源码分析文章，没错，我就只写了一个简单的 demo 就开始看别人的源码分析了，当时还挺兴奋的，毕竟之前自己都没怎么看过别人项目的源码，看了源码之后才发现自己是有多菜，源码基本看不懂，即使是经过别人分析再写得浅显易懂的解释自己也看得一脸懵逼，到后来硬撑着看完所有文章之后发现自己好像啥都没学到。
  • docker，k8s，rancher。对于云原生这一套东西当时公司是明确后续要使用到的，所以去学习了下。docker 的使用倒是了解了，但是对于 k8s 无论是概念还是使用基本都不怎么会，当时看文档我甚至都没能动手打一个 demo，对于 rancher 也就只会在网页上点点点。自己对于云原生这一套的学习还是很欠缺，甚至是云原生这三个字都不怎么理解。
  • 剩下的还有 minio、zookeeper、postgresql、mongodb、kkfileview、领域驱动设计等等，不过这写都知识零零散散的学了一点，了解的地方不多。
  • 9月份换地方住之后关注了很多公众号，在地铁上偶尔会翻翻上面的文章，不过这些文章对于我来说感觉就像是在看小说，看过就忘。

# 计划

多看书，多写代码，多做笔记。

  • 多看书是

因为这半年来学的东西都太过于零散，没有做到系统的学习，导致很多学过的东西都很容易忘记。

  • 多写代码

这方面主要是觉得自己代码写得太少了，工作上就不说了，学习上虽然学一个新知识会做一个简单的 demo，但是感觉距离实际的应用上还是有些差距的。

  • 多做笔记

这半年学的东西不多，笔记做的更少，一方面是因为自己没找到一个好写笔记的地方，另一方面就是懒了。
